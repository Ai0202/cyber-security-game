# CyberGuardians — 体験型セキュリティ教育ゲーム

## 概要

CyberGuardians は、「攻撃者の目線で学ぶ、サイバーセキュリティ体験学習」をコンセプトとした体験型の情報セキュリティ教育ゲームである。従来の座学中心のセキュリティ研修では実感が湧きにくいという課題を解決するため、プレイヤーが**攻撃者（ATTACK SIDE）**の視点を体験できるインタラクティブなゲームを提供する。

プレイヤーは「ストーリー（攻撃テーマ）」を選び、4つのフェーズ（情報収集→認証窃取→侵入→目的達成）で構成される攻撃チェーンを体験する。各フェーズでは複数の攻撃手法「コンポーネント」からランダムに選ばれたミニゲームをプレイし、同じストーリーでも毎回異なる体験が得られる。

攻撃シナリオは Gemini API によって毎回動的に生成され、ストーリーのコンテキスト（業界・ターゲット組織）に合わせた一貫性のある体験を提供する。

## ターゲットユーザー

### メインターゲット
- 一般の会社員（IT部門以外を含む）
- セキュリティ研修の受講者

### サブターゲット
- 主婦・主夫（SNS経由の詐欺被害が増加傾向）
- 学生（SNSからの個人情報流出リスク）
- 情報セキュリティに興味はあるが専門知識がない方全般

## 技術スタック

- フロントエンド: Next.js (App Router) + TypeScript + Tailwind CSS
- アニメーション: Framer Motion
- AI: Google Gemini API（シナリオ動的生成・AIフィードバック・チャット対話）
- データ定義: JSONファイル（`src/data/`）。将来的にRDBに移行
- 状態管理: React Context（クライアントサイド）
- デプロイ: Google Cloud Run

## コンセプト

- **「攻撃を知ることが、最大の防御になる」** をテーマに、攻撃者の手口を体験して防御力を高める
- プレイヤーは「シャドウ」というハッカーキャラクターを操作して、ストーリーに沿った攻撃チェーンを体験する
- ストーリーを選ぶと、4フェーズ × ランダムコンポーネントで毎回異なるプレイ体験
- 攻撃体験後、各フェーズでの防御策・学習ポイントを提示
- サイバーパンク風のかっこいいUI（ダーク基調、ネオンカラーのアクセント）で没入感を演出
- Gemini APIを活用し、ストーリーコンテキストに合わせた動的シナリオ生成

## ゲーム構造

```
ストーリー選択
  └─ ストーリー（攻撃テーマ）
      ├─ Phase 1: 情報収集（RECONNAISSANCE）
      │   └─ コンポーネント（ランダム選択）: ショルダーハッキング / ダンプスターダイビング / ...
      ├─ Phase 2: 認証窃取（CREDENTIAL THEFT）
      │   └─ コンポーネント（ランダム選択）: フィッシング / パスワードクラッキング / ...
      ├─ Phase 3: 侵入（INTRUSION）
      │   └─ コンポーネント（ランダム選択）: ネットワーク侵入 / ...
      └─ Phase 4: 目的達成（OBJECTIVE）
          └─ コンポーネント（ランダム選択）: ランサムウェア / データ窃取 / ...
```

## スコープ

### MVP（本プロジェクトの対象）

- **ストーリーモード**
  - 1つ以上のストーリー（攻撃テーマ）
  - 4フェーズの攻撃チェーン
  - フェーズ間のコンテキスト引き継ぎ
- **コンポーネント（攻撃手法ミニゲーム）**
  - ショルダーハッキング（情報収集）
  - パスワードクラッキング（認証窃取）
  - フィッシング（認証窃取）
  - ネットワーク侵入（侵入）
  - ランサムウェア展開（目的達成）
- **練習モード**: コンポーネント単体でのプレイ
- **スコアリング + ランク判定**: 各コンポーネント 0〜100点、S/A/B/C/D ランク
- **セッション結果画面**: AIによるフィードバック・防御策の解説
- **データ定義**: JSONファイルでストーリー・コンポーネントを管理

### 将来の拡張（MVP対象外）

- ユーザー認証・プロフィール
- RDBへのデータ移行
