# App 1: サイバーセキュリティ体験学習ゲーム

## 概要

ユーザーが攻撃者となり、企業のネットワークに侵入する体験を通じて、サイバー攻撃の手口・損害・防御策を学ぶWebゲーム。コンピュータやネットワーク機器を擬人化したキャラクターが登場し、親しみやすい世界観で展開する。

### ターゲットユーザー

| ユーザー | 利用シーン | ニーズ |
|---------|-----------|-------|
| 一般従業員 | セキュリティ研修 | 楽しく学べる研修。「やらされる感」がない |
| 情シス担当者 | 研修管理 | 受講率・理解度の可視化、監査対応 |
| 経営層 | 導入判断 | ISMS/Pマーク対応の研修実績 |

### 設計思想

- **「攻撃者の立場を体験することで、守り方を学ぶ」** が核心思想
- 受動的に「これは危険です」と教わるのではなく、自分が攻撃を実行する体験を通じて「こんなに簡単に突破できてしまうのか」という危機感を醸成する
- 擬人化キャラクターが攻撃で苦しむ姿を見せることで、感情移入を通じた学習効果を狙う

---

## 世界観

プレイヤーの勤める会社が「サイバーシティ」というデジタル世界として描かれる。

- オフィスビル = サーバー群
- 社員 = データ
- 警備員 = セキュリティソフト

## ゲームモード

### モードA: 攻撃者体験（Attack Side）

プレイヤーが攻撃者「シャドウ」を操作し、企業への侵入を試みる。

#### Stage 1: ショルダーハッキング

- **場面**: カフェでノートPCを開いて仕事をしている社員を観察
- **操作**: 視点を動かして画面を覗き込む、付箋に書かれたパスワードを見つける、社員証の情報を読み取る
- **演出**: 発見すべき情報漏洩ポイントが3箇所あり、タップで発見。全発見でクリア
- **学び**: 公共の場でのPC操作リスク、覗き見防止フィルターの重要性、パスワードの物理管理

#### Stage 2: パスワードクラッキング

- **場面**: 盗んだハッシュ値に対して様々な攻撃を実行
- **操作**: 辞書攻撃・ブルートフォース・パスワードリスト攻撃を選択して実行
- **演出**: パスワードの強度によって突破時間がリアルタイム表示される
  - 「password」→ 0.001秒
  - 「tanaka1985」→ 3分
  - 「Xk#9pL!2qW$m」→ 推定380年
- **演出**: キャラクター「パスワ」の姿が弱いPWだとヨロヨロ、強いPWだと屈強に変化
- **学び**: 弱いパスワードがいかに瞬殺されるか、多要素認証の重要性

#### Stage 3: フィッシング攻撃

- **場面**: 偽メールを作成してターゲット社員に送信する
- **操作**: 差出人偽装、本物そっくりのログインページ作成、緊急性を煽る文面の作成
- **演出**: キャラクター「メーラ」がうっかり開いてしまう場面を見る
- **学び**: フィッシングメールの見分け方、URL確認の習慣

#### Stage 4: ランサムウェア侵攻

- **場面**: 侵入に成功した後、社内ネットワークを横展開
- **操作**: キャラクター「クリプト」の暗号化能力を悪用して、ファイルを次々と暗号化
- **演出**:
  - サーバー内のファイルが緑（安全）→ 赤（暗号化済み）に変化するグリッドアニメーション
  - サーバーキャラが苦しむ姿
  - 身代金要求画面の表示
  - 「バックアップン」に復旧を頼むと復元される
- **学び**: ランサムウェアの仕組み、感染拡大のメカニズム、バックアップの重要性

#### Stage 5: ソーシャルエンジニアリング

- **場面**: 電話やメールで社員から情報を聞き出す
- **操作**: AIチャットで社員役と対話し、機密情報を引き出す
- **演出**: 相手の警戒レベルゲージが表示され、怪しまれるとゲームオーバー
- **学び**: 情報の聞き出しテクニック、正規の確認手順の重要性

#### Stage 6: 公衆Wi-Fi中間者攻撃

- **場面**: カフェの偽Wi-Fiアクセスポイントを設置
- **操作**: 偽APを設置し、接続してきた人の通信を傍受する様子を観察
- **学び**: 公衆Wi-Fiのリスク、VPN利用の重要性

### モードB: 防御者体験（Defense Side）

攻撃者体験で学んだ知識を使って、今度は会社を守る側に回る。

- **パスワード強化チャレンジ**: 「パスワ」を鍛えてレベルアップさせる
- **フィッシング仕分けゲーム**: 大量のメールから詐欺メールを見分ける。制限時間内にスワイプで判定
- **インシデント対応シミュレーション**: ランサムウェア感染発覚後の初動対応を体験。ネットワーク遮断→報告→バックアップ復旧の手順
- **セキュリティポリシー構築**: 予算内でセキュリティ対策を選択・配置する経営シミュレーション

### モードC: 対戦・協力モード

- **レッドチーム vs ブルーチーム**: チーム対抗で攻撃側・防御側に分かれて対戦
- **部署対抗ランキング**: 月次のセキュリティクイズで部署別スコアを競う
- **インシデント協力プレイ**: チームで分担してサイバー攻撃に対応する協力ゲーム

## AI活用ポイント

### 1. ソーシャルエンジニアリング対話AI

- **用途**: Stage 5でLLMが社員役を演じ、自然な会話の中で警戒レベルを変動させる
- **技術**: Claude API（ロールプレイ）
- **仕様**:
  - AIは社員役として自然に応答する
  - ユーザーの発言内容に応じて「警戒レベルゲージ」を0〜100で変動
  - 怪しい質問をすると警戒レベルが上昇し、100に達するとゲームオーバー
  - 巧みに情報を引き出せると成功
- **具体例**: 「IT部門の者ですがパスワードリセットの確認で…」→ AIが怪しみつつも応答、警戒ゲージが上昇

### 2. フィッシングメール動的生成

- **用途**: モードBの「フィッシング仕分けゲーム」で、毎回異なるフィッシングメールをAIが生成
- **技術**: Claude API（テキスト生成）+ テンプレートエンジン
- **仕様**:
  - 正規メールとフィッシングメールをAIが動的に生成
  - パターン暗記では対応できない訓練にする
  - 難易度に応じて「見分けにくさ」を調整
- **具体例**: 「社長からの緊急メール」「配送業者の不在通知」「税務署からの還付通知」など毎回変化

### 3. 個人・部署別セキュリティ弱点分析

- **用途**: プレイ結果をAIが分析し、個人や部署単位での弱点と改善提案を自動生成
- **技術**: Claude API（分析）+ 統計処理
- **仕様**:
  - 各ステージのスコア・回答パターンを集計
  - 「フィッシング検知が弱い」「パスワード意識が低い」などカテゴリ別に弱点を特定
  - 個人向け改善アドバイスと、部署向け傾向レポートを生成
- **具体例**: 「営業部はフィッシング検知率が全社平均より20%低い。特にURLチェックの習慣が不足」

## ゲーミフィケーション

### レベルシステム

```
Lv.1: セキュリティ見習い
Lv.2: パスワード戦士
Lv.3: フィッシングハンター
Lv.4: インシデントレスポンダー
Lv.5: サイバーガーディアン（称号獲得）
```

### バッジシステム

- 🛡️ **鉄壁のパスワード**: パスワードステージをSランククリア
- 🎣 **フィッシング検知マスター**: 50通連続で正解
- 🔥 **インシデント即応**: 対応シミュレーションを3分以内にクリア
- 👁️ **ショルダーハック防御**: ショルダーハッキングステージで全ての手口を発見
- 🏆 **全社1位**: 月次ランキングでトップ

### 継続施策

- 毎月新シナリオ追加（最新の攻撃手口を反映）
- 四半期ごとの全社イベント（チーム対抗戦）
- スコアをセキュリティ研修の修了証として利用可能

## 安全性・リスク対策

| リスク | 対策 |
|--------|------|
| 攻撃手法の悪用 | 原理の理解に留め、実行可能なツールや手順は提供しない。常に「こう防ぐ」に帰結させる |
| LLMの不適切出力 | 教育目的に限定したシステムプロンプト + 出力フィルタ + 人間レビュー体制 |
| 「ゲーム=不真面目」という認識 | ISMS監査対応レポート機能で公式研修としての価値を証明 |
| 研修が一度きりで終わる | 月次新シナリオ + ランキングリセット + バッジコレクションで継続モチベーション設計 |
| 高齢社員がゲームUIに馴染めない | 「クラシックモード」として従来型のクイズ形式も用意 |
